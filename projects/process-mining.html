Project 03
Process mining · KU Leuven

Business Process
Mining & BPMN

Applied Celonis process mining to real-world event log data. Performed variant analysis, throughput time analysis and resource workload analysis across 847 process instances. Identified efficiency bottlenecks and modelled end-to-end workflows using BPMN.

Tools: Celonis · BPMN · Process mining
Context: KU Leuven Business Analysis
Cases: 847 process instances
01 — Overview
The problem

Process performance was measured mostly through manual reporting and anecdotal feedback. There was no systematic visibility into how often processes deviated from the intended flow, where delays accumulated, or which activities were overloaded.

The goal was to use process mining on real event log data to surface objective evidence of how the process actually ran — not how it was supposed to run — and translate those findings into concrete improvement recommendations.

02 — Methodology
Analytical approach

Event log data was imported into Celonis for process discovery and conformance checking. Three analytical angles were used: variant analysis (comparing actual process flows to the intended flow), throughput time analysis (identifying where time was lost), and resource workload analysis (identifying overloaded roles or activities).

Findings from the quantitative analysis were then used to construct a BPMN process model that documented both the happy path and the exception flows, with annotations marking bottleneck activities.

04 — Results
Key findings
33% deviated from happy path
Mainly due to incomplete input data triggering manual review. Pre-validation check at submission estimated to reduce rework by ~60%.
2.1-day bottleneck at approval
Manual approval was the primary throughput driver. Automating approvals for standard cases would reduce median case duration from 4.2d to 2.1d.
Monday/Tuesday overloaded
40% higher case volumes at the start of the week. Workload smoothing through staggered submission windows would reduce peak-day backlog.
Top 5 variants cover 91% of cases
High variant concentration means a few targeted process fixes would cover the vast majority of deviation cases — low effort, high impact.
